**C. Replace loadSubTasks function**

Replace the existing `loadSubTasks` function:
```typescript
async function loadSubTasks() {
  subTasksLoading = true;
  try {
    taskTree = await getTaskWithSubtasks(params.id);
    // Auto-expand root's direct children
    if (taskTree?.sub_tasks?.length) {
      expandedNodes = new Set([params.id]);
    }
  } catch {
    taskTree = null;
  } finally {
    subTasksLoading = false;
  }
}
```

**D. Add tree toggle helper**

Add a helper function to toggle node expansion:
```typescript
function toggleNode(nodeId: string) {
  const next = new Set(expandedNodes);
  if (next.has(nodeId)) {
    next.delete(nodeId);
  } else {
    next.add(nodeId);
  }
  expandedNodes = next;
}
```

Add a Svelte 5 `{#snippet}` block in the `<script>` area or just before the template where snippets are valid. This renders a single tree node and recurses for children:


**Important implementation notes:**
- The `{#snippet}` / `{@render}` syntax is Svelte 5's replacement for recursive components. It works like a template function.
- The `style="padding-left: {depth * 20}px"` provides visual indentation per tree level.
- Root-level children (depth 0) start with no indentation. Each deeper level adds 20px.
- The expand/collapse toggle only shows for nodes that have children. Leaf nodes get a spacer for alignment.
- `expandedNodes` is a reactive Set — toggling triggers re-render since we replace the entire Set.
- Keep the existing `subTasks` removal clean — remove the old `subTasks` state variable entirely since it's replaced by `taskTree`.
