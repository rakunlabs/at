**1. `func (s *Server) runOrgDelegation(ctx context.Context, org *service.Organization, task *service.Task, agentID string, depth int) error`**

This is the core recursive delegation function. It:

a) **Enforces depth limit**: If `depth >= org.MaxDelegationDepth`, update task status to "completed" with result "max delegation depth reached" and return nil. Use 10 as default if `org.MaxDelegationDepth` is 0.

b) **Loads the agent**: `s.agentStore.GetAgent(ctx, agentID)`. If not found, update task to "completed" with error result and return.

c) **Resolves provider**: `s.getProviderInfo(agent.Config.Provider)`. If not found, update task to "completed" with error result and return.

d) **Gets direct reports**: Call `s.getDirectReports(ctx, org.ID, agentID)` (helper below). These are the agent's potential delegates.

e) **Builds delegate tools**: For each direct report (active status only), create a `delegate_to_{safename}` tool using the EXACT same name-sanitization pattern from `agent-call.go` (see interfaces above). Store a map of `toolName → reportAgentID` for dispatch.

f) **Builds enriched system prompt**: Start with `agent.Config.SystemPrompt`. Append a section:
```
## Your Team (Direct Reports)
You can delegate tasks to these team members using the delegate_to_* tools:

- {agent.Name} ({orgAgent.Role}, {orgAgent.Title}): {agent.Config.Description}
```
For each direct report, look up the Agent record to get Name and Config.Description. If the agent has no direct reports (leaf node), skip the team section.

g) **Updates task status** to "in_progress" via `s.taskStore.UpdateTask`.

h) **Runs agentic loop** (follow the `chat-sessions.go` pattern exactly):
- Max iterations from `agent.Config.MaxIterations` (default 10 if 0).
- Before each LLM call: check budget via `s.checkBudgetFunc()` — if budget exceeded, update task to "completed" with budget error result, return nil.
- Call `provider.Chat(ctx, model, messages, llmTools)`.
- After each call: record usage via `s.recordUsageFunc()`.
- Record audit via `s.recordAuditFunc()` for each tool call (same pattern as agent-call.go lines 695-712).
- Build messages array with system prompt, user message (task.Title + "\n\n" + task.Description), and tool call/result exchange.
- Strip Handler/HandlerType from tools before sending to LLM (same as agent-call.go lines 554-561).

i) **Tool call dispatch**:
- For `delegate_to_*` tools: extract `task` argument. Create child task via `s.createDelegationTask(ctx, org, task, reportAgentID, taskText, depth)`. Then recursively call `s.runOrgDelegation(ctx, org, childTask, reportAgentID, depth+1)`. Return the child task result (or error) as the tool result string.
- For unknown tools: return "Error: unknown tool {name}" (don't crash).

j) **On completion** (LLM returns without tool calls): Update task status to "completed", set `task.Result` to the final LLM response. Return nil.

**2. `func (s *Server) getDirectReports(ctx context.Context, orgID, agentID string) ([]service.OrganizationAgent, error)`**

List all org agents via `s.orgAgentStore.ListOrganizationAgents(ctx, orgID)`, filter to those where `ParentAgentID == agentID` and `Status == "active"`. Return the filtered list.

**3. `func (s *Server) createDelegationTask(ctx context.Context, org *service.Organization, parentTask *service.Task, assigneeAgentID, description string, depth int) (*service.Task, error)`**

- Increment issue counter: `s.organizationStore.IncrementIssueCounter(ctx, org.ID)`
- Build identifier using org.IssuePrefix (fallback to first 4 chars of org.ID) + "-" + counter
- Create task: `s.taskStore.CreateTask(ctx, service.Task{...})` with:
  - OrganizationID = org.ID
  - ParentID = parentTask.ID
  - AssignedAgentID = assigneeAgentID
  - Title = parentTask.Title (keep the original title for traceability)
  - Description = description (the task text from the delegate tool call)
  - Status = TaskStatusOpen
  - Identifier = generated identifier
  - RequestDepth = depth + 1

**Important patterns to follow:**
- Error handling: always wrap with `fmt.Errorf("context: %w", err)` per project convention
- Logging: use `slog.*` structured logging with descriptive messages
- All log messages prefixed with "org-delegation:" for grep-ability
- File name convention: hyphen-separated (`org-delegation.go`)
- Import grouping: stdlib → third-party → internal
- The delegation is synchronous within the goroutine — the recursion happens inline, not as separate goroutines. Concurrent fan-out is Phase 3 (CONC-01). For Phase 2, if the LLM makes multiple delegate tool calls in one response, process them sequentially.
