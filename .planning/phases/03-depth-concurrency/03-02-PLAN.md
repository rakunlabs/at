  <action>
Add `"sync"` to the import block in `org-delegation.go`.

Replace the sequential tool call loop (the section starting with the comment `// i) Execute tool calls sequentially (concurrent fan-out is Phase 3).` through the end of the `for _, tc := range resp.ToolCalls` loop, ending just before `// Append tool results and continue loop.`) with a concurrent version:

```go
// i) Execute tool calls concurrently (fan-out: one goroutine per delegation).
toolResults := make([]service.ContentBlock, len(resp.ToolCalls))
var wg sync.WaitGroup
var resultMu sync.Mutex

for i, tc := range resp.ToolCalls {
    slog.Debug("org-delegation: tool call",
        "tool", tc.Name, "task_id", task.ID, "iteration", iteration)

    if reportAgentID, ok := delegateToolMap[tc.Name]; ok {
        wg.Add(1)
        go func(idx int, toolCall service.ToolCall, targetAgentID string) {
            defer wg.Done()

            taskText, _ := toolCall.Arguments["task"].(string)
            if taskText == "" {
                taskText = task.Title
            }

            childTask, err := s.createDelegationTask(ctx, org, task, targetAgentID, taskText, depth)
            if err != nil {
                resultMu.Lock()
                toolResults[idx] = service.ContentBlock{
                    Type:      "tool_result",
                    ToolUseID: toolCall.ID,
                    Content:   fmt.Sprintf("Error: failed to create delegation task: %v", err),
                }
                resultMu.Unlock()
                return
            }

            slog.Info("org-delegation: delegating to report",
                "parent_task", task.ID, "child_task", childTask.ID,
                "from_agent", agentID, "to_agent", targetAgentID, "depth", depth+1)

            var result string
            if delegErr := s.runOrgDelegation(ctx, org, childTask, targetAgentID, depth+1); delegErr != nil {
                result = fmt.Sprintf("Error: delegation failed: %v", delegErr)
            } else {
                updated, getErr := s.taskStore.GetTask(ctx, childTask.ID)
                if getErr != nil {
                    result = fmt.Sprintf("Delegation completed but failed to fetch result: %v", getErr)
                } else if updated != nil && updated.Result != "" {
                    result = updated.Result
                } else {
                    result = "Delegation completed (no result returned)."
                }
            }

            resultMu.Lock()
            toolResults[idx] = service.ContentBlock{
                Type:      "tool_result",
                ToolUseID: toolCall.ID,
                Content:   result,
            }
            resultMu.Unlock()

            // Record audit entry for delegation tool call.
            recordAudit := s.recordAuditFunc()
            if recordAudit != nil {
                auditDetails := map[string]any{
                    "tool_name": toolCall.Name,
                    "task_id":   task.ID,
                    "iteration": iteration,
                    "has_error": false,
                }
                if auditErr := recordAudit(ctx, service.AuditEntry{
                    ActorType:      "agent",
                    ActorID:        agentID,
                    Action:         "tool_call",
                    ResourceType:   "tool",
                    ResourceID:     toolCall.ID,
                    OrganizationID: org.ID,
                    Details:        auditDetails,
                }); auditErr != nil {
                    slog.Warn("org-delegation: failed to record audit",
                        "agent_id", agentID, "error", auditErr)
                }
            }
        }(i, tc, reportAgentID)
    } else {
        // Unknown tool — handle synchronously (no goroutine needed).
        toolResults[i] = service.ContentBlock{
            Type:      "tool_result",
            ToolUseID: tc.ID,
            Content:   fmt.Sprintf("Error: unknown tool %q", tc.Name),
        }

        // Record audit for unknown tool call.
        recordAudit := s.recordAuditFunc()
        if recordAudit != nil {
            auditDetails := map[string]any{
                "tool_name": tc.Name,
                "task_id":   task.ID,
                "iteration": iteration,
                "has_error": true,
            }
            if auditErr := recordAudit(ctx, service.AuditEntry{
                ActorType:      "agent",
                ActorID:        agentID,
                Action:         "tool_call",
                ResourceType:   "tool",
                ResourceID:     tc.ID,
                OrganizationID: org.ID,
                Details:        auditDetails,
            }); auditErr != nil {
                slog.Warn("org-delegation: failed to record audit",
                    "agent_id", agentID, "error", auditErr)
            }
        }
    }
}

wg.Wait()
```

**Key design decisions:**
- Pre-allocate `toolResults` as indexed slice (not append) — each goroutine writes to its own index, so the mutex is only needed for safety, not ordering.
- All goroutines share the same `ctx` (derived from `context.Background()` via task-intake.go). Do NOT use `errgroup.WithContext` — we want all delegations to complete even if one fails, because the LLM expects results from ALL tool calls.
- Unknown tools are handled synchronously inline (no goroutine needed, they're instant).
- Audit recording moved inside each goroutine so it happens alongside the delegation, not after.
- The `iteration` variable is captured by closure (safe — it's loop-local and not mutated during goroutine execution).

Also update the comment at the top of the agentic loop section to remove the Phase 3 deferral note.

**Note on test strategy:** Full runOrgDelegation tests require mocking LLM providers which is complex. Instead, test the key behaviors:

1. **TestConcurrentDelegation** — Test that `createDelegationTask` can be called concurrently (the store operation is what matters for race conditions):
   - Create a server with mock stores
   - Pre-populate: org "org1" with head agent "A" and 3 direct reports "B", "C", "D"
   - In a goroutine fan-out pattern (mimicking the production code), call `createDelegationTask` 3 times concurrently for the same parent task
   - Use `sync.WaitGroup` to coordinate
   - Run with `-race` flag
   - Assert all 3 child tasks are created with correct ParentID
   - Assert no races detected (the -race flag handles this)

2. **TestConcurrentDelegationResults** — Test that tool results from concurrent goroutines are correctly collected into the indexed slice:
   - Create a pre-allocated `[]service.ContentBlock` slice of length 3
   - Launch 3 goroutines, each writing to its own index (with mutex, matching production pattern)
   - `wg.Wait()`
   - Assert all 3 results are populated (non-empty Content)
   - Assert order matches index (result[0] has toolCall[0]'s ID, etc.)

3. **TestDeepDelegation** — Test `createDelegationTask` at multiple depths to verify ParentID chaining:
   - Create parent task P, then child C1 from P (depth 0), then grandchild C2 from C1 (depth 1), then great-grandchild C3 from C2 (depth 2)
   - Assert C1.ParentID == P.ID, C2.ParentID == C1.ID, C3.ParentID == C2.ID
   - Assert C1.RequestDepth == 1, C2.RequestDepth == 2, C3.RequestDepth == 3
   - This proves the recursive delegation chain creates correct task linkage at arbitrary depth
